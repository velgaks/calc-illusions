# Методологія

## Мета

Оцінити частку (і абсолютну кількість) людей на підконтрольній території України, які відповідають набору критеріїв одночасно. Критерії: стать, вік, зріст, дохід, освіта, куріння, алкоголь, діти, шлюб.

## Чому joint-розподіл, а не множення маргінальних

Якщо взяти частку чоловіків (≈48%), помножити на частку з ВО (≈40%), помножити на частку з доходом вище медіани (50%), помножити на частку некурців (≈70%) і т.д. — отримаємо число, яке припускає **незалежність ознак**. Але:
- Освіта корелює з доходом
- Освіта корелює з курінням (негативно)
- Дохід корелює з віком
- Шлюб корелює з наявністю дітей

Множення дає **систематичне заниження** частки людей у престижних/освічених стратах і **завищення** у комбінаціях, що зустрічаються рідко. Виправлення — рахувати спільну частку безпосередньо з мікроданих репрезентативного опитування.

## Дані

### ESS — Європейське соціальне дослідження (мікродані)

Основне джерело joint-розподілу демографічних і поведінкових змінних.

- Джерело: [europeansocialsurvey.org](https://www.europeansocialsurvey.org/)
- Хвиля: ESS Round 11 (поле 2023–2024), Ukraine
  - Fallback: ESS Round 10 (поле 2020), якщо R11 для України виявиться надто обмеженою
- Вибірка: репрезентативна для дорослого населення (15+) території, де проводилось поле
- **Зважування:** `pspwght` (post-stratification weight) або `anweight` (analytic weight) — усі операції зважені, без винятків

### Зріст — NCD Risk Factor Collaboration

ESS не вимірює зріст. Використовуємо параметричну модель за статтю і когортою народження.

- Джерело: [ncdrisc.org](https://ncdrisc.org/) (Lancet, 2020 і пізніші)
- Що беремо: μ і σ зросту у см для дорослих, окремо чоловіки і жінки, по 5-річних когортах народження
- Розрахунок: P(зріст ≥ h | стать, когорта) = 1 − Φ((h − μ)/σ), де Φ — стандартна нормальна CDF
- Для діапазону віків — зважене середнє μ і σ за чисельністю когорт

### Чисельність когорт — Інститут демографії ім. Птухи НАН України

Перетворення частки в абсолютну кількість.

- Джерело: [demography.org.ua](https://www.demography.org.ua/)
- Що беремо: оцінки чисельності за статтю і 5-річними групами для підконтрольної території
- Цільова сума по всіх групах: ≈30.5 млн (наявне населення, не міжнародно визнані кордони)

### Дохід — Держстат, ОУЖД

Конвертація «дохід від X грн» → дециль для запиту до joint-розподілу ESS.

- Джерело: [ukrstat.gov.ua](https://www.ukrstat.gov.ua/) → Обстеження умов життя домогосподарств
- Що беремо: межі дохідних децилів у грн/міс на одну особу
- Дисклеймер: ESS питає дохід **домогосподарства**, ОУЖД — на **особу**. Конвертація через середній розмір домогосподарства або через перерахунок ESS hinctnta на per capita — спірне місце, фіксуємо явно в `methodology.json`

## Змінні ESS, які використовуємо

| Змінна | Опис | Перекодування |
|---|---|---|
| `gndr` | Стать (1=чол, 2=жін) | прямо |
| `agea` | Точний вік на момент опитування | прямо |
| `eisced` | Освіта ISCED 2011 (1–7) | прямо |
| `hinctnta` | Дохідний дециль домогосподарства (1–10) | прямо |
| `marsts` | Шлюбний стан | бінаризуємо: одружений/решта |
| `chldhm` | Діти у домогосподарстві (1=так, 2=ні) | бінаризуємо |
| `cgtsmke` | Частота куріння | бінаризуємо: курить/не курить |
| `alcfreq` | Частота вживання алкоголю | категорія: ніколи/рідко/часто |
| `rlgdgr` | Релігійність 0–10 | бінаризуємо: ≥5 vs <5 |
| `lrscale` | Політика left-right 0–10 | бінаризуємо: ≤4 vs ≥6 (центр 5 — ні те, ні те) |
| `dosprt` | К-сть днів за 7, коли була фіз. активність ≥30 хв | бінаризуємо: ≥3 vs ≤2 |
| `lnghom1` | Мова вдома (ISO 639-3) | категорія: ukr / rus / other |
| `pspwght` | Post-stratification weight | вага для усіх агрегацій |
| `anweight` | Analytic weight (якщо є у R11) | альтернативна вага |

Якщо змінної у фактичному файлі немає або вона перейменована — фіксуємо в `public/data/methodology.json` як `variables_missing` і прибираємо відповідний контрол з UI. Не вигадуємо fallback.

## Алгоритм

```
1. (стать, віковий діапазон) →
   N_base = Σ external.cohorts[sex][ageBucket].population

2. (дохід у грн/міс) →
   incomeDecileMin = найменший дециль з межею ≥ дохід_користувача
   через external.incomeDeciles

3. Joint-частка з ESS:
   window = respondents.filter(r.gndr == sex && r.agea ∈ [ageMin, ageMax])
   denom  = Σ window.pspwght
   match  = window.filter(r → satisfies всі ESS-критерії користувача)
   numer  = Σ match.pspwght
   joint_share = numer / denom

4. P(зріст ≥ h) — параметрично з NCD-RisC, weighted average по когортах у віковому вікні

5. share_final = joint_share × p_height
   count_final = share_final × N_base

6. Bootstrap CI (95%):
   500 resampleів з window з заміщенням
   → quantile [0.025, 0.975] на joint_share
   Множення на детермінований p_height розширює CI пропорційно
```

## Confidence intervals

Простий bootstrap не враховує survey design (страти, кластери). Це **зміщує SE вниз** — істинне CI ширше за показане. Для MVP приймаємо.

У R-stage додатково обчислюємо design-aware SE через `survey::svymean`. Результат записується у `respondents.json` як `precomputed_se` — фронтенд може опціонально перемикатися на «design-adjusted CI».

## Decomposition

«Який критерій звужує найсильніше» — leave-one-out:
- Для кожного активного критерію c обчислюємо joint-частку без c
- Δ_c = (share_without_c − share_with_c) / share_without_c
- Сортуємо за спаданням

Це наївна декомпозиція (не Shapley). Інтуїтивно інтерпретована: «якщо прибрати критерій X, твоя частка зросте до Y%».

## Обмеження

1. **Підконтрольна територія** ≠ міжнародно визнані кордони. Усі чисельні оцінки — наявне населення, де можливе фактичне дослідження.
2. **ESS — обмежена вибірка.** Для тонких розрізів (наприклад, чоловіки 18–22 з ISCED 7) може бути 0–3 респонденти. CI вибухає. UI попереджає коли n_window у comparisonі < 30.
3. **Зріст — Normal-модель.** Не враховує селекційні ефекти (висока мортальність у нижніх перцентилях зросту тощо). Систематичне зміщення малоймовірне.
4. **Дохід.** ESS `hinctnta` — дециль на час опитування (2023–2024). Інфляція з тих пір — додаткове джерело шуму. Це фіксуємо в `methodology.json`.
5. **Час.** Опитування — точкова оцінка на момент поля. Демографія, доходи, поведінка змінюються. Дисклеймер «дані станом на YYYY» — у бейджі під результатом.

## Що це НЕ

- НЕ статистично точна оцінка ймовірності існування конкретної людини
- НЕ оцінка індивідуального шансу зустріти таку людину
- НЕ заміна перепису населення
- НЕ комерційний або політичний інструмент

Це інструмент **методологічного контрасту**: показати, наскільки наївне перемноження викривлює реальність, і дати чесну альтернативу з відкритими джерелами і CI.
