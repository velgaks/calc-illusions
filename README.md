# Скільки таких українців?

🟢 **Live:** https://velgaks.github.io/calc-illusions/  
📦 **Repo:** https://github.com/velgaks/calc-illusions

Постав свої ознаки (стать, вік, освіта, дохід, мова, погляди, фізичні параметри) і дізнайся скільки таких людей живе на підконтрольній території. На основі 2581 інтерв'ю ESS Ukraine 2024.

Інструмент оцінює частку людей на підконтрольній території України, які відповідають заданим критеріям (стать, вік, зріст, дохід, освіта, звички).

**Принципова відмінність від популярних калькуляторів-агрегаторів:** замість наївного перемноження маргінальних часток (що дає, наприклад, 0.0001%, бо «припускає незалежність» там, де її немає) ми працюємо із joint-розподілом на мікроданих [ESS Ukraine](https://www.europeansocialsurvey.org/) зі зваженням `pspwght`. Зріст і чисельність когорт — окремі множники з власними джерелами.

> Статистика — не вирок. Але й не повід вигадувати ймовірності з повітря.

## Стек

- **Препроцесинг:** R (`haven`, `survey`, `dplyr`, `jsonlite`)
- **Фронтенд:** React + Vite, без бекенду
- **Дані для браузера:** 3 статичні JSON-и у `public/data/`
- **Деплой:** GitHub Pages

## Структура

```
calculator/
├── data/raw/        # сирі .sav (НЕ комітимо)
├── prep/            # R-pipeline: .sav → JSON
├── public/data/     # 3 JSON для браузера
└── web/             # React-додаток
```

Детально — у [METHODOLOGY.md](./METHODOLOGY.md).

## Швидкий старт (на mock-даних)

```bash
cd web
npm install
npm run dev
```

Відкриється на `http://localhost:5173` з банером «DEMO DATA — дані синтетичні». Це нормально для розробки фронтенду. Перехід на реальні ESS-дані — [`RUN_REAL.md`](./RUN_REAL.md).

## Як це працює

1. `prep/build.R` читає `.sav`, перекодовує змінні, рахує joint-розподіл зі зваженням, експортує `respondents.json` (≈1500 рядків з реальною ESS-вибіркою).
2. Браузер тягне JSON-и одноразово. Для кожного користувацького запиту фільтрує respondents по статі/віку, рахує зважену joint-частку, множить на коефіцієнт зросту (NCD-RisC).
3. Bootstrap-CI рахується на льоту з 500 ресемплів. Design-aware SE з R-stage також доступний.

## Принципи

- **Не вигадуємо числа.** Немає джерела — TODO в коді і явний дисклеймер в UI.
- **Не беремо з вторинних джерел.** Тільки ESS portal, ukrstat.gov.ua, demography.org.ua, ncdrisc.org.
- **Не ігноруємо ваги ESS.** Усе зважене групування.
- **Фіксуємо територію.** Тільки підконтрольна, з прямим disclaimerом.
